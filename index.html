第二站：基础概念

AI 世界最基本的 4 块积木，后面所有概念都建立在它们之上：LLM、Token、Context、Prompt

入门

LLM — 大语言模型，也就是AI的大脑

Large Language Model

LLM 就像一个读过几乎全部图书馆书籍的超级学霸。它学会了人类语言的规律，能理解你说的话并生成新的文字来回答。

生活比喻：一个百科全书式的朋友，你问什么它都能聊，但它不能上网查新消息，也不能帮你订外卖——它只会"说话"。
深入了解 LLM →
入门

Token — 词元

Token

Token 是 LLM 读写文字的最小单位。一个中文字约 1-2 个 token，一个英文单词约 1-3 个 token。LLM 不直接看"字"，而是把文字切成 token 来处理。

生活比喻：拼乐高——一整句话被拆成一块块积木（token），AI 一块一块地读，再一块一块地拼出回答。
AI 怎么"切"文字

Context — 上下文窗口

Context Window

Context 是 LLM 一次能"看到"的全部内容，包括之前的对话历史记录、系统提示和当前用户的新提问等。

每次问大模型问题，就相当于把你的问题、之前的聊天记录、系统提示词、网络搜集的关键内容等全都打包丢给 AI 去处理，然后 AI 根据内容返回答案。

Prompt — 提示词

Prompt

Prompt 就是你对 AI 说的话。写得好，AI 回答得聪明；写得模糊，AI 可能答非所问。"提示词工程"就是研究怎么把话说清楚的学问。

生活比喻：点菜——你说"随便"，厨师可能做什么都有；你说"不辣的清淡鱼"，厨师就知道该怎么做了。

第四站：Agent 与 Multi-Agent

LLM 只能"说话"，Agent（智能体）能"做事"——从嘴巴升级为双手。

了解

Agent — 智能体

AI Agent

Agent = LLM + 工具 + 规划能力。它不仅能思考，还能自己决定下一步做什么：搜索网页、写代码、读文件、调用 API……

生活比喻：LLM 像一个坐在椅子上的学者，只能回答问题。Agent 像一个能走来走去的助手，不仅能回答你，还会主动帮你查资料、写邮件、订机票。
Anthropic Agent 文档 →
了解

Multi-Agent — 多智能体

Multi-Agent System

多个 Agent 组成团队协作，各有分工。比如一个 Agent 负责搜索、一个负责写代码、一个负责测试，像一个小公司一样运转。

生活比喻：一个人做完整个项目很累。但有人负责搜资料、有人负责写文档、有人负责检查——各司其职，合在一起就是 Multi-Agent 团队。

热门

OpenClaw — 开源 AI Agent 新星

Open-Source Personal AI Agent

OpenClaw 是 2026 年初爆火的开源 AI Agent 框架，由 PSPDFKit 创始人 Peter Steinberger 打造，GitHub 星标超 24 万。它是一个本地运行、全天在线的私人 AI 助手，通过 Telegram、WhatsApp、Discord 等聊天软件和你对话，背后可以接 Claude、GPT、DeepSeek 等各种大模型。

核心亮点：

  • 本地优先 — 数据存在你自己的设备上，隐私有保障
  • 100+ AgentSkills — 执行命令、管文件、自动化浏览网页等
  • 多模型支持 — 自由选择底层 LLM，本地部署可零成本运行
  • 主动触达 — 不用你开口，它可以主动通知你（如日程提醒）
生活比喻：如果 ChatGPT 是一个在线客服，那 OpenClaw 就是住在你家里的 AI 管家——24 小时待命，什么事都能帮你跑腿。

第五站：进阶概念

让 AI 更加强大的几个关键"装备"。

Skill — 技能

Skill / Tool

Skill 是 Agent 能使用的具体能力，比如"搜索网页""生成图片""执行代码"。每一个 Skill 是一个小工具，Agent 根据任务选择合适的 Skill。

生活比喻：Agent 是万能工匠，Skill 是工具箱里的各种工具——锤子、螺丝刀、电钻。看到钉子拿锤子，看到螺丝拿螺丝刀。

MCP — 模型上下文协议

Model Context Protocol

MCP 是 Anthropic 提出的开放标准，让 AI 模型能用统一方式连接各种外部工具和数据源。可以理解为 AI 世界的"USB 接口"。

生活比喻：以前每种手机都有自己的充电线，现在统一用 Type-C。MCP 就是 AI 工具连接的 Type-C——不管什么工具，只要支持 MCP，AI 就能直接用。
MCP 官方文档 →

RAG — 检索增强生成

Retrieval-Augmented Generation

LLM 的知识可能过时或不完整。RAG 的做法是：先从知识库检索相关资料，把资料塞进 Prompt，再让 LLM 生成回答。

生活比喻：开卷考试！学生（LLM）做题时可以翻书（检索知识库）。闭卷只靠记忆容易出错；开卷有参考资料，答案更准确。

RAG 流程：用户提问 → 系统检索知识库 → 把相关文档片段 + 问题放入 Prompt → LLM 根据资料生成答案

第八站：AI 发展时间线

这些概念和公司是怎么一步步出现的。

2017

Transformer 诞生

Google 发表《Attention Is All You Need》，提出 Transformer 架构，成为所有现代 LLM 的基石。

2018-2020

GPT 系列起步

OpenAI 发布 GPT-1 → GPT-2 → GPT-3，模型越来越大，能力越来越强。

2020

RAG 概念提出

Meta 研究院发表 RAG 论文，提出"先检索后生成"的思路。

2021

Anthropic 成立

OpenAI 前研究副总裁等人创立 Anthropic，专注 AI 安全研究。

2022.11

ChatGPT 横空出世

两个月用户破亿，AI 从实验室走进千家万户。

2023.3

GPT-4 与 Claude 发布

大模型竞争进入白热化阶段。

2023

Agent 概念爆发

AutoGPT、BabyAGI 等项目掀起热潮，探索让 AI 自主完成复杂任务。

2024.1

DeepSeek 崛起

以极低训练成本达到顶级水平，震动全球 AI 行业。

2024.11

MCP 协议发布

Anthropic 发布 Model Context Protocol，为 AI 工具连接建立统一标准。

2025

Multi-Agent 时代

多智能体协作日益成熟，Claude CodeX 等 Agent 产品改变开发者工作方式。
